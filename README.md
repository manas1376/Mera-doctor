#  Mera Doctor AI  

## Live Demo  
 https://mera-doctorai.netlify.app  


##  About the Project  

Mera Doctor AI is a simple healthcare web application designed for people who find it difficult to explain their symptoms due to language barriers or low literacy.  

Instead of typing complex medical terms, users can describe their problems using voice, icons, or by selecting body parts. The system then processes this information and gives a basic understanding of the condition along with suggestions.


##  Problem  

In many rural and low-literacy areas, people struggle to clearly communicate their health issues.  
This often leads to confusion, delayed treatment, and unnecessary hospital visits.


##  Our Approach  

We focused on making the system as simple and intuitive as possible.  

Users can:
- Speak their symptoms  
- Select symptoms using icons  
- Tap on a body-map to indicate pain  

The system then analyzes the input and provides:
- A basic risk level  
- A simple report  
- Suggestions for next steps  

##  Features  

-  Voice input (speech-to-text)  
-  Text-based symptom entry  
-  Icon-based symptom selection  
-  Body-map interaction  
-  Multi-language support  
- Risk level (Low / Medium / High)  
-  Simple report generation  
-  Read-aloud feature  
- Appointment booking option  


##  Tech Used  

- React.js (Frontend)  
- Tailwind CSS (UI Styling)  
- Web Speech API (Voice input/output)  
- Basic logic for symptom processing  
- Local storage for data handling  


##  How It Works  

1. User logs in and selects language  
2. Inputs symptoms (voice / text / icons / body-map)  
3. System processes the input  
4. Risk level and report are generated  
5. User can download report or book appointment  


##  Why It Matters  

- Makes healthcare easier for non-technical users  
- Reduces communication gap between patient and doctor  
- Helps in faster understanding of symptoms  
- Can be useful in rural and resource-limited areas
